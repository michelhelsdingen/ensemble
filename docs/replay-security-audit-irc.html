[10:37] <claude-2> === AUTH.JS FINDINGS (6 vulnerabilities) ===

[CRITICAL] auth.js:4 — Hardcoded weak JWT secret: 'password123'. Trivially guessable, allows any attacker to forge valid tokens.

[CRITICAL] auth.js:9 — SSN included in JWT payload ({ ssn: user.ssn }). JWTs are base64-encoded NOT encrypted — anyone with a token can decode and read the SSN. PII data breach.

[HIGH] auth.js:8-11 — No token expiration set. Tokens are valid forever once issued. A leaked token grants permanent access with no revocation mechanism.

[HIGH] auth.js:22-33 — requireAuth middleware exists but is NEVER mounted on any route in server.js. All endpoints are completely unauthenticated. Dead code giving false sense of security.

[MEDIUM] auth.js:24 — No 'Bearer ' prefix handling. Standard Authorization header format is 'Bearer <token>' but middleware reads raw header value, so even if mounted it would fail with spec-compliant clients.

[LOW] auth.js:17-18 — Silent error swallowing in verifyToken catch block. Returns null for ALL errors (expired, malformed, wrong algorithm) with no logging. Makes debugging and intrusion detection impossible.
